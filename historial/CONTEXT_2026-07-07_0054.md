## 1. VISIÓN Y META

### Visión
El objetivo es generar la mayor rentabilidad posible, en el menor tiempo posible — pero siempre ajustada al riesgo.

No se trata de disparar más trades sino de disparar los correctos. Cada decisión debe tener una razón técnica sólida, un riesgo controlado, y una expectativa matemática positiva demostrada con datos reales.

El camino hacia eso es un bot completamente autónomo que aprende de sus propios resultados, mejora con cada trade sin intervención manual, y opera 24/7 con disciplina profesional — sin emociones, sin excepciones, sin improvisar.

### Hoja de ruta
- **Fase 1 (ahora):** 200+ trades reales con contexto técnico completo — la base de datos de la que aprenderá todo lo demás
- **Fase 2:** Clasificador supervisado — solo entra cuando la probabilidad de éxito es alta y el riesgo está justificado
- **Fase 3:** Parámetros adaptativos — SL, T1 y trailing se ajustan automáticamente según las condiciones del mercado
- **Fase 4:** Agente RL autónomo — descubre y optimiza estrategias completas desde la experiencia acumulada

### 4.1 Parámetros (V5.0 — heredados del simulador validado)
```
Pares:          SOL/USDT:USDT | ETH/USDT:USDT  (ma=SMA)   ← se QUITÓ k_lim=45 (ver §4.11)
STOP_LOSS:      -0.85%
BE_TRIGGER:     +0.15%   → activa break-even
BE_STOP:        +0.10%   → cierre si retrocede con BE activo (cubre comisión 0.05% entrada + 0.05% salida)
T1_TARGET:      +0.25%   → cierra 50% de la posición
SL_REAL_PCT:    0.85%    → stop_market en Binance al abrir (seguro ante crash del proceso)
C1 piso ATR:    ATR_15M >= 0.45%   → no entra si la volatilidad 15M está por debajo (corrección nueva)
C2 time-stop:   45 min   → cierra la posición si no resolvió en 45 min (corrección nueva)
C3 entrada:     limit post-only al mejor bid, timeout 60s → si no llena, descarta (NO_FILL). Comisión 0.04% maker vs 0.10% taker (corrección nueva)
BLOQUEOS:       post-STOP y post-RETROCESO_BE (dos bloqueos tras pérdida)
Escaneo:        3s libre / 0.5s en posición (heredado del motor del sim)
Capital/trade:  ~95% del balance disponible
```

### 4.2 Trailing post-T1 (2 niveles, V5.0 — del sim)
```
Ganancia:          Trailing aplicado:
0.00% → 0.40%     ganancia - 0.20%
>= 0.40%          ganancia - 0.10%  (aprieta)
Suelo siempre:    max(SUELO_POST_T1 = T1 = 0.25%, trailing_stop)
trailing nunca retrocede (solo sube)
```

### 4.3 Filtros de entrada (V5.0 — del sim, SIN K<45) + C1
**Paso 1 — motivo_entrada:**
```
SNIPER   si pendiente_7 > 0.10% AND vol_r > 1.30x
ESTANDAR si block == "" AND pendiente_7 > 0.10%   (SNIPER tiene prioridad)
```
**Paso 2 — filtros generales:**
```
pasa_btc  = btc_trend_suav != 'BAJISTA'   ← V5 usa el SUAVIZADO (el sim usaba btc_trend directo); el campeón lo espeja
pasa_15m  = slope_ema5_15m > 0.05%
gap_ok    = gap200 <= 4.0%
C1        = atr_15m >= 0.45%               ← piso de volatilidad (corrección nueva)
```
**Paso 3 — F1-F5:**
```
F1 vol      = vol_r >= 0.30x
F2 k15m     = k_mayor_d_15m OR k_d_diff_15m > -5   ← ahora SÍ evalúa (en el sim viejo era no-op por bug de keys; efecto ~neutro, K casi no separa)
F3 combo    = NOT (ESTANDAR + ASIATICA + btc ALCISTA)
F4 bloqueo  = símbolo no en bloqueo post-STOP / post-RETROCESO_BE
F5 funding  = NOT (funding_alto AND motivo == ESTANDAR)
```
**Lo que se QUITÓ del cerebro V4:** el filtro estocástico `K < 45`, que chocaba con el momentum (K bajo + pendiente alta casi nunca alinean) → el bot real V4 hizo solo 7 trades entre marzo y mayo. Ver §4.11.

### 4.4 Secuencia de gestión de posición (V5.0, + C2 time-stop)
**Con posición abierta:**
1. Actualizar g_max / g_min
2. Si `t1_hecho`: trailing 2 niveles → si toca → **CIERRE_TRAILING** (vende restante)
3. Si ganancia >= T1 (0.25%) y no t1_hecho → **T1** (vende 50%, recoloca SL real, activa trailing con suelo)
4. Si ganancia >= BE_TRIGGER (0.15%) y no be_activado → **BE_ACTIVADO** (solo registra)
5. Si no t1_hecho (salidas):
   - be_activado y ganancia <= BE_STOP (0.10%) → **RETROCESO_BE** (vende 100%)
   - no be_activado y ganancia <= STOP_LOSS (-0.85%) → **STOP_LOSS** (vende 100%)
   - precio < ema7 y no be_activado → **CORTE_EMA7** (vende 100%)
6. **C2 (nuevo):** si la posición lleva > 45 min sin resolver → **TIME_STOP** (cierra 100%)

**Sin posición (no ocupado):**
1. Verificar bloqueos post-STOP / post-RETROCESO_BE
2. Filtros (§4.3) → si pasan: **ENTRADA por limit post-only** al mejor bid, timeout 60s. Si llena → coloca SL real. Si NO llena en 60s → **NO_FILL** (descarta, sigue escaneando).

"K>{k_lim}"      si k > k_lim AND pendiente_7 < 0.10% (solo si pendiente baja)
"K<D"            si k < d AND pendiente_7 < 0.10% (solo si pendiente baja)
"P<EMA7"         si precio < ema7
"PLANA"          si pendiente_7 <= 0.05%
"FUNDING_ALTO:{rate}" si funding_alto (informativo; el filtro lo maneja el bot según tipo de señal)

## 10. ANÁLISIS ESTADÍSTICO

### 10.A Validación de la estrategia — 496 trades simulados (W15→W23) ← EL QUE DECIDIÓ EL TRANSPLANTE
Análisis completo de la estrategia del simulador sobre **496 trades válidos** (todos en régimen **bajista**), evaluando **PnL NETO** (con comisión) y validado **fuera de muestra (OOS)**. Documento completo: `analisis/ANALISIS_ESTRATEGIA.md`. Hallazgos centrales — lectura honesta:

- **En bruto la estrategia es plana** (≈ −0.01%/trade); **neta pierde** ≈ −0.110%/trade.
- **La comisión es el ~91% de la pérdida.** De ahí C3 (limit/maker, 0.10% → 0.04%), la palanca más grande — con el riesgo de que baje la tasa de llenado.
- **Perfil 77% WR con asimetría 1:3.7** (gana muchas chicas, las pocas pérdidas son grandes) → "diseñada para empatar, no para ganar".
- **Correlación con el mercado 0.13:** no monta tendencias. T1 corta a +0.25% mientras pasan movimientos de 5-7%.
- **Único separador ganador/perdedor: la volatilidad (ATR), y es débil** → de ahí C1 (piso ATR_15M ≥ 0.45).
- **Correcciones validadas:** C1 (piso ATR), C3 (limit/maker), C2 (time-stop 45min). El combo **R2 = C1+C2+C3** lleva la estrategia a **breakeven** en bajista.
- **Conclusión honesta (transmitida a Juan):** hoy la estrategia PIERDE; el mejor caso realista es **dejar de perder** (breakeven en bajista). De dónde sale el EDGE positivo es la **pregunta abierta no resuelta** (candidata: Fase 2, un clasificador entrenado con datos propios — ver §13).
